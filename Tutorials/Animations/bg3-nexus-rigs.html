Working with the IK Rigs

written by kit

In this section of the animation tutorials we will go over using these body rigs in Blender for the purpose of making poses/animations.

1. Setup

Please take the time to refer to the setup section of the import-export article.

2. About the rigs

These rigs were set up using Auto-Rig Pro, a paid Blender addon. It is however not necessary to have it to use the rigs at a base level. While having it provides some useful additional functionality for making animations and retargeting animations, it is not at all required for making static poses.

Within the master .blend file, the rigs are organised into collections for all the playable body types/races. Each body type has a control rig (CR) and a skeleton rig (SK). The control rig is meant for animating on, and serves a puppeteering function for the skeleton rig. The skeleton is where you would bake the animation in the end and what you export out of Blender back into the game.

As of writing this tutorial, all of the rigs are facing in the positive Y direction, and are mirrored, so the limbs you perceive to be on the left are named right, and so on. This is intentional (and is due to Blender’s axis mismatch with the game’s engine), and any animation you make on them will be mirrored in-game, so keep that in mind.

Why the bones on the skeletons are oriented like That™

Bones in Granny/GR2 files used in BG3 actually function more like joints, akin to Maya etc., not Blender, where a bone has a head and a tail. A joint only occupies one point in space compared to two by a bone head and tail. Joints are interpreted as bone heads by Blender.

Another difference is that the primary local axis (the axis that goes through/along the length of the joint/bone) of a bone in Maya/GR2 is X, whereas in Blender it is Y. When importing a joint-based GR2 file with the addon, Blender essentially guesses the position of the bone tail based on its own standard for a primary axis. Because it is different in Blender, visually the bones appear to be "sideways". This is meant to be this way and will be correctly exported back into the game by the addon, since the position of the tails does not technically matter (but the position/orientation of the heads does matter).

TL;DR: Do not worry, they are meant to look exactly like that and will be exported back correctly into the game.

IK-FK limb functions

For this explanation, you can think of a character’s limb, e.g. arm, as a chain of bones: shoulder → elbow → wrist.

For ease of posing/animating, the limbs on all the rigs in the .blend file are set to IK by default. Switching between IK and FK (and vice-versa) on the go is known as IK-FK snapping. Unless you are doing some in-depth animating, or are retargeting an animation to the control rigs (more on that here), you would not need to switch between them; however, here is how to do it with these rigs for posterity:

If you have the Auto-Rig Pro Rig Tools addon:

With an end of your limb (e.g. the arm control bone) selected in Pose Mode, locate the Rig Main Properties menu (by default it would be under the Tools tab in your side panel), and use the Snap IK-FK button.
