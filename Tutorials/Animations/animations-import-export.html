Animations Import-Export

written by kit

In this section of the animation tutorials we will go over importing BG3 poses/animations and exporting your poses/animations from Blender.

1. Setup

Required software and Add-ons

Blender – recommended version is 4.5+ (3.6 is the minimum compatible version with the addon below).

Norbyte's BG3/DOS2 Collada Exporter - An addon for Blender that allows you to import and export GR2 files for Baldur’s Gate 3 through either Collada (.DAE) or GLTF. It is always best to check for new commits and update accordingly.

Lslib – a tool by Norbyte for manipulating Baldur’s Gate 3 files. It is always best to check for new releases and update (and set a new path to your updated Divine.exe) accordingly.

Animation Template with BG3 IK Animation Rigs – a collection of control rigs (head and body) ready for animating.

Within Blender

For the addon to be able to import and export your animations straight to/from GR2 (via GLTF or Collada), configure the path to Divine.exe within your addon preferences. You will find Divine.exe under the Tools folder in a lslib release.

2. Animations Import

Find an animation/pose of your choice. For the purposes of this article, we will be using HUM_F_Rig_SCENE_CRA_PlayerFreefall_Loop.GR2 (the freefall animation during the nautiloid crash for BT1 Human).

GR2 animations/poses, particularly vanilla game animations, are "raw" animation transform values without any skeleton data within them. Therefore, to import them into Blender with a skeleton, you must copy a _Base skeleton (corresponding to the required body type) via LSLIB first.

Browse to and import your source GR2 animation, and then specify the path to HUM_F_Base.GR2 (since we are using BT1 here) within your unpacked files. Specify an output path (it can also be a GR2, or even the same path as the input file to overwrite it with a skeleton-containing version). Leave options like “Mirror Skeletons” and “Mirror Meshes” unchecked.

Hit “Export”. Your output GR2 can then be imported into Blender with the aforementioned addon. 

Do NOT use import via Collada for importing animations, Collada messes with the orientations of the GR2 bones/joints on import and will leave your animation unsuitable for export back into the game.

Additionally, your animations will import with keyframes scaled according to the FPS you had set in Output Properties in your .blend file (by default it is 24 FPS). It is best to set it to either 30 FPS or 60 FPS before import, as this is the FPS the animations usually are in the game to preserve the original keyframes.
